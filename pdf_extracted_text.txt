3.1. Algorithmic Framework The proposed framework aims to reduce the actionability gap in refactoring tools by moving beyond simple code smell detection toward prioritized refactoring recommendation. Instead of only identifying potential smells, the system combines static metrics, detector -style smell signals, Git-history indicators, model-backed confidence, and dependency-aware sequencing to help developers decide which refactoring candidates may be more important to review first. This approach is intended to support more informed and structured refactoring decisions, while still requiring developer validation, compilation, and testing before any change is considered complete. 3.1.1.
